1、基本概述

二分算法的基本用法是在一个单调有序的集合或函数中查找一个解。因此，当问题的答案具有单调性时，可以考虑将问题的解，通过二分，转化为某种判定问题。每次分成左右两个区间，通过判断调整上下界，即继续搜索左区间还是右区间，直到找到目标元素。

更进一步的说，还可以扩展到三分算法取解决峰值函数的极值问题。三分算法将区间分成 3 个部分，可以明确判断答案不在哪一部分，舍弃三分之一的空间，继续进行查找。

此外，还有浮点数上的二分，其与整数二分大同小异。

总结一下，一般初学者在练习时可能会碰到以下问题：

  • 整数集上的二分，需要注意左右区间的开闭情况，避免漏掉答案或者死循环；
  • 实数域上的二分，需要注意精度问题。

二分的实现方法灵活多样，主要是处理边界上的差异时需要仔细考虑。为了避免混淆写法，下面只着重介绍一种实现方法。

2、算法实现

2.1、整数集上的二分

若a[mid]<xa[mid] < xa[mid]<x，可知此时midmidmid 一定不是我们的解，故l=mid+1l = mid + 1l=mid+1，抛弃左半段。

可见这个二分写法有两种形式：

  1. r=mid,l=mid+1r = mid,l=mid + 1r=mid,l=mid+1，对于中间值，有mid=(l+r)/2mid = (l + r) / 2mid=(l+r)/2；
  2. l=mid,r=mid−1l = mid, r = mid - 1l=mid,r=mid−1，对于中间值，有mid=(l+r+1)/2mid = (l + r + 1) / 2mid=(l+r+1)/2。

为什么要区分midmidmid 的形式呢？

主要是因为要避免死循环。假如让形式2为mid=(l+r)/2mid = (l + r) / 2mid=(l+r)/2，那么当lll 与rrr 相邻时，就有mid=⌊l+r2⌋=lmid = \lfloor\frac{l+r}{2}\rfloor = lmid=⌊2l+r​⌋=l，若接下来进入l=midl = midl=mid 分支，就会造成死循环；若进入r=mid−1r = mid-1r=mid−1 这个分支，则循环退出时l>rl > rl>r，答案不在区间内。所以这样区分是有必要的。

这种写法比较灵活，其好处是循环退出的条件是l=rl = rl=r，也就是不用考虑最后的答案到底是在左端点还是右端点。

为了进一步加深理解，我们观察这两种形式，不难发现：主要是查找到两个相邻的两个值时，先去判断左端点还是右端点，前者先判断的是左端点，后者先判断的是右端点。所以我们可以利用这个特点，来决定具体采用那哪种形式。

例如，如果我们要找的是最大的一个解，那么在二分查找的过程中，区间会不断缩小，直至两个端点相邻。此时，先去判断较大的那个端点rrr，如果rrr 不满足条件，就让r=mid−1r = mid - 1r=mid−1，也就是说排除这个右端点，否则就让l=midl = midl=mid。如果要找的是最小的一个解也是同理。

此外，我们也可以利用这一特点，发现：形式1不会取到rrr，形式2不会取到lll。因此，我们可以扩展原来的区间[1,n][1,n][1,n] 到[1,n+1][1,n+1][1,n+1] 和[0,n][0, n][0,n]。扩大边界的好处是：如果目标不在我们查找的区间中，则最后位置一定在我们额外扩充的边界上，此时说明区间内不存在解。

总结一下吧，这种写法的基本流程是：

  1. 通过分析问题，确定对于 mid，下一次要查找的是左半段还是右半段，以及 mid 保留在哪一个半段；
  2. 根据分析结果，确定 mid 是用哪种形式，相邻时先判断右端点还是左端点；
  3. 循环退出的条件是l=rl = rl=r，所以最后答案任意输出其中一个即可。

这里提一下 C++ STL 中的两个函数 upper_bound 和 lower_bound，分别是查找第一个大于xxx 的元素和第一个大于等于xxx 的元素。

这里总结了以下情况：

  1. 查找第一个大于xxx 的元素的位置：upper_bound()。例如，pos = upper_bound(a, a + n, x) - a；
  2. 查找第一个大于等于xxx 的元素的位置：lower_bound()；
  3. 查找第一个与xxx 相等的元素：lower_bound() 且=x=x=x；
  4. 查找最后一个与xxx 相等的元素：upper_bound() 的前一个且=x=x=x；
  5. 查找最后一个小于等于xxx 的元素：upper_bound() 的前一个；
  6. 查找最后一个小于xxx 的元素：lower_bound() 的前一个。

2.2、实数域上的二分

也叫浮点二分，与整数集上的二分的写法大同小异，主要考虑精度问题。一般题目会给出保留小数位数或误差范围kkk。定义一个 EPS 常量为 1e-12（具体根据题目设置），循环退出的条件由原来的l<rl < rl<r 改为l+eps<rl + eps < rl+eps<r，midmidmid 不区分形式，只有mid=(l+r)/2mid = (l + r) / 2mid=(l+r)/2。

2.3、三分求凸性函数极值

凸性函数即在区间内是单峰函数，有唯一的极大值或极小值点（只能选其一）。

实现的思路是：对于一个开口向下的函数fff,在定义域[l,r][l, r][l,r] 内，取两个点lmidlmidlmid 和rmidrmidrmid，从而将整个函数分成了三段。

  1. 如果f(lmid)<f(rmid)f(lmid) < f(rmid)f(lmid)<f(rmid)，则lmidlmidlmid 和rmidrmidrmid 同时在极大值的左侧，或是一个在左一个在右。无论是哪种情况，lmidlmidlmid一定是在极大值的左侧，所以让l=lmidl = lmidl=lmid；
  2. 如果f(lmid)>f(rmid)f(lmid) > f(rmid)f(lmid)>f(rmid)，则lmidlmidlmid 和rmidrmidrmid 同时在极大值的右侧，或是一个在左一个在右。无论是哪种情况，rmidrmidrmid 一定是在极大值的右侧，所以让r=rmidr = rmidr=rmid；
  3. 通过上述的两个过程，区间不断往极大值逼近，最终求的解。

3、例题讲解

3.1、整数集上的二分

3.1.1、寻找指定和的整数对

问题描述

输入nnn 个整数，找出其中的两个数，它们的和等于mmm。

题解

采用暴力的做法是用嵌套循环，枚举出所有的情况，时间复杂度为O(n2)O(n^2)O(n2)。

采用二分算法优化查找。首先要确保序列的单调性，所以对数组进行排序。在查找的过程中，假设当前的数字是aia_iai​，查找在该数字后是否存在一个数字的值是m−aim - a_im−ai​ 即可。因为数组是单调的，所以可以用二分优化查找第一个等于m−aim - a_im−ai​ 的值。排序直接用 STL 中的 sort 函数，所以最后整个算法的复杂度是O(nlog⁡n)O(n\log n)O(nlogn)。

练习：P1102 A-B 数对 - 洛谷(luogu.com.cn)

3.2、实数集上的二分

3.2.1、小车问题

问题描述

两人一车，人比车快，求它们从 A 到 B 的最短时间。

题解

假设甲、乙两人甲先坐车，乙走路。

不难想到最短的时间是两人同时出发，车先带甲到半路，然后让甲自己走到终点，车子再回去接乙，最终两人同时到达终点。此时花费的时间就是最少的时间。所以问题的关键是找到这个车子折返点。

简单分析一下，如果折返点距离起点越远，那么乙花费的时间就越长；如果折返点距离起点越近，那么甲花费的时间就越长。因此可以发现距离xxx 与花费时间ttt 有单调关系。因此可以用二分来解决。

具体做法是枚举xxx，然后分别计算甲、乙到达终点的时间，如果甲先到达，那么就缩短其坐车的时间，即r=midr = midr=mid；否则就增加其坐车的时间，即l=midl = midl=mid。最终计算出两者到达终点时间相同，就是最短的时间。时间复杂度为O(log⁡n)O(\log n)O(logn)。

练习：P1258 小车问题 - 洛谷 (luogu.com.cn)

3.3.1、Balanced Stone Heaps

问题描述

有nnn 堆石头，每堆石头有hih_ihi​ 个石头。你可以通过以下操作该改变每堆石头的数量：

  • 从第333 堆石头开始操作，直到第nnn 堆石头；
  • 假设当前是第iii 堆石头，可以选择ddd 个石头移到第i−1i-1i−1 堆，同时2d2d2d 个石头移到第i−2i-2i−2 堆，所以如果这样操作之后，第iii 堆减少了3d3d3d 个石头。
  • 每次操作ddd 满足0≤3d≤hi0\le 3d\le h_i0≤3d≤hi​。

求操作之后最少石头的那堆的最大值。

题解

对最小高度二分。对于当前假设的高度xxx，每次操作的时候，采取贪心策略，倒序枚举，把当前的石头数尽可能地放到前两堆。需要注意的是，当前堆石头数减少的数量一定是000 或是333 的正整数倍数。

练习：Problem - 1623C - Codeforces
